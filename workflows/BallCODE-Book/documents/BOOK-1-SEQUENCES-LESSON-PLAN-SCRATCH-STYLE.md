## â±ï¸ LESSON STRUCTURE (45-60 minutes)

### Part 1: Introduction (5 minutes)

**Hook:**
"Have you ever given someone directions? 'First, go straight. Then, turn left. Finally, stop at the corner.' That's a sequence - one step after another. Today, we're going to learn how to give directions to a computer using sequences!"

**Connection to Basketball:**
"Just like giving directions, basketball plays are sequences. First, you move. Then, you pass. Finally, you shoot. Let's see how this works in code!"

**Learning Goal:**
"Today, we're learning about sequences - the order of steps in code."

---

### Part 2: Story Reading (10 minutes)

**Read:** Book 1, Episode 1 - "The Foundation Block"

**Key Moments to Pause:**

**After Act I:**
- "What sequence of moves did the player use?"
- "What happened first? Then what? Finally what?"

**After Act II:**
- "How did the player break down the press?"
- "What was the order of moves?"

**After Act III:**
- "How did understanding sequences help win the game?"

**Discussion Questions:**
1. What is a sequence? (One thing after another)
2. Why does order matter? (Can't shoot before you have the ball)
3. How is this like giving directions? (Step by step)

---

### Part 3: Skill Pit-Stop (5 minutes)

**Mini-Lesson: Sequences**

**Visual Demonstration:**
Draw on whiteboard:
```
1. Move Forward
2. Pass Ball
3. Move to Position
```

**Explain:**
- "A sequence is steps in order"
- "Code executes one step at a time"
- "Order matters - you can't pass before you move"

**Connect to Basketball:**
- "Breaking the press is a sequence"
- "Each move happens in order"
- "If you skip a step, it doesn't work"

---

### Part 5: Game Exercise (15 minutes)

**Exercise: Breaking the Press**

**Instructions:**
1. Open the game exercise
2. Create a sequence to break the press
3. Use these blocks: START, MOVE FORWARD, PASS BALL, MOVE TO POSITION
4. Test your sequence
5. Debug if needed

**Scratch-Style Guidance:**
- Start simple: Just 2-3 blocks
- Test it: See if it works
- Add more: Build complexity
- Debug: Fix what doesn't work

**Success Criteria:**
- âœ… Sequence has at least 3 blocks
- âœ… Blocks are in logical order
- âœ… Player successfully breaks the press
- âœ… Student can explain the sequence

**Differentiation:**
- **Struggling:** Provide starter sequence, students add one block
- **On Track:** Students create their own sequence
- **Advanced:** Students create complex sequence (5+ blocks)

## ğŸ’¡ TEACHER TIPS

### Scratch-Style Teaching Tips

**1. Start Simple:**
- Don't overwhelm with too many blocks
- Start with 2-3 blocks
- Build complexity gradually

**2. Visual First:**
- Show blocks, don't just explain
- Use whiteboard/projector
- Let students see it work

**3. Hands-On:**
- Students must try it themselves
- Don't just demonstrate
- Let them experiment

**4. Immediate Feedback:**
- Game shows results immediately
- Students see if it works
- Easy to debug

**5. Celebrate Mistakes:**
- "What did you learn from that?"
- "How can we fix it?"
- "Debugging is part of coding!"
