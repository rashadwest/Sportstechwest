## üìã PART 1: CLEAR FRAMEWORK ANALYSIS

### C - Clarity: What is SMART-PED?

**SMART-PED Framework Breakdown:**
- **S** - **Specify the Role:** Define AI's role (e.g., "Act as an experienced university faculty teaching AI")
- **M** - **Mention the Purpose:** State the educational goal (e.g., "Lesson plan," "Assessment," "Research support")
- **A** - **Define the Audience:** Identify learners (e.g., "First-year," "Postgraduate," "Slow learners")
- **R** - **Pedagogical Constraints:** Set educational parameters (e.g., "Bloom's levels," "CO alignment," "Real-world examples")
- **T** - **Output Format:** Specify structure needed (e.g., "Table," "PPT bullets," "Rubric")
- **PED** - **Prompt ‚Üí Evaluate ‚Üí Develop:** Iterative refinement cycle (e.g., "Refine: Simplify," "Add activity," "Improve engagement")

**Key Message:** "Effective prompting = Clear role + purpose + pedagogy + structure"

### E - Examples: How Would We Use SMART-PED?
