# Video Recording Strategy: 3 Focus Areas
## Using IBM SkillsBuild Teacher Toolkit as Baseline

**Purpose:** Create video recordings tailored to three different student learning preferences  
**Baseline:** IBM SkillsBuild Teacher Toolkit structure and best practices  
**Approach:** Three distinct video styles for maximum engagement

---

## The Three Focus Areas

### 1. **Love Basketball** üèÄ
**Student Profile:** Passionate about basketball, plays/watch games, knows players/teams  
**Entry Point:** Basketball excitement and action  
**Learning Style:** Kinesthetic, visual, sports-focused

### 2. **Studies Basketball** üìä
**Student Profile:** Analytical, interested in stats/strategy, data-driven thinking  
**Entry Point:** Basketball analysis, statistics, game theory  
**Learning Style:** Logical, analytical, detail-oriented

### 3. **Interested in Stories** üìñ
**Student Profile:** Loves narratives, characters, adventure, storytelling  
**Entry Point:** Story-driven learning, character development, plot

## IBM SkillsBuild Baseline Structure

### Core Components (From IBM Toolkit):
1. **Clear Learning Objectives** - What students will learn
2. **Engaging Introduction** - Hook that captures attention
3. **Structured Content** - Organized, progressive learning
4. **Interactive Elements** - Activities, questions, engagement
5. **Real-World Connections** - Career relevance, practical application
6. **Assessment/Reflection** - Check understanding, reinforce learning
7. **Extension Activities** - Deeper exploration, career connections

### Video Structure (Adapted from IBM):
- **Hook/Introduction** (1-2 min)
- **Core Content** (10-15 min)
- **Interactive Checkpoint** (1-2 min)
- **Application/Example** (3-5 min)
- **Career Connection** (2-3 min)
- **Wrap-Up/Next Steps** (1-2 min)

**Total Duration:** 18-27 minutes per episode

## Focus Area 1: Love Basketball üèÄ

### Video Style & Approach

**Tone:** Energetic, exciting, action-packed  
**Pacing:** Fast, dynamic, keeps energy high  
**Visuals:** Game footage, highlights, court action, player movements  
**Language:** Basketball terminology, game scenarios, real plays

### Structure for "Love Basketball" Videos

#### 1. Hook/Introduction (1-2 minutes)
**Goal:** Get them excited about basketball action

**Content:**
- Start with exciting basketball moment (game highlight, buzzer beater, amazing play)
- "Have you ever watched a game and wondered how players make split-second decisions?"
- "Today we're going to learn how coding works by breaking down that exact play!"
- Show the play/moment that connects to the coding concept

**Example (Episode 1 - State Management):**
- Show a fast break transition play
- "Watch how the game state changes: START (tip-off) ‚Üí LIVE (fast break) ‚Üí OUTCOME (score)"
- "That's exactly how code tracks what's happening!"

#### 2. Core Content (10-15 minutes)
**Goal:** Teach coding concept through basketball action

**Approach:**
- Use real game scenarios
- Break down actual plays
- Show basketball examples first, then connect to code
- Use basketball terminology throughout
- Show court diagrams, play breakdowns
- Use game footage to illustrate concepts

**Content Organization:**
- **Basketball Example First** (3-5 min)
  - Show a real play/scenario
  - Break it down step-by-step
  - Use basketball language
  
- **Connect to Coding** (3-5 min)
  - "This is exactly how code works!"
  - Show the coding concept
  - Map basketball ‚Üí code
  
- **Practice with Basketball** (3-5 min)
  - More basketball examples
  - Practice identifying the concept in plays
  - Interactive: "What state is this?"

#### 3. Interactive Checkpoint (1-2 minutes)
**Goal:** Check understanding with basketball scenarios

**Content:**
- "Let's test what you learned!"
- Show 2-3 basketball scenarios
- Ask: "What's the state here?" or "What would the code do?"
- Pause for student response
- Reveal answer with explanation

#### 4. Application/Example (3-5 minutes)
**Goal:** Show how it works in real games

**Content:**
- Show multiple game examples
- Break down different scenarios
- "Here's how this works in different situations"
- Use actual game footage when possible
- Connect to famous plays/players (if appropriate)

#### 5. Career Connection (2-3 minutes)
**Goal:** Connect to tech careers through basketball

**Content:**
- "Did you know game developers use this exact concept?"
- Show how video games track player states
- "Software engineers use state management in apps"
- "Frontend developers use this for websites"
- Show real examples (video game UI, app states)
- Basketball connection: "Just like tracking game state, apps track user state"

#### 6. Wrap-Up/Next Steps (1-2 minutes)
**Goal:** Reinforce learning, preview next steps

**Content:**
- Quick recap: "We learned [concept] through basketball"
- "Try watching a game and tracking the states!"
- Preview next episode
- Call to action: "Complete the exercises to practice"

### Recording Tips for "Love Basketball" Style

**Energy:**
- High energy, enthusiastic
- Sound excited about basketball
- Use dynamic voice inflection
- Match the excitement of the game

**Visuals:**
- Use game footage liberally
- Court diagrams with action
- Play breakdowns
- Player movements highlighted
- Fast-paced editing

**Language:**
- Basketball terminology first
- "Possession," "transition," "fast break"
- Then connect to code terms
- Keep basketball language primary

**Examples:**
- Real game scenarios
- Famous plays (if appropriate)
- Current players/teams (if appropriate)
- Action-packed moments

## Focus Area 2: Studies Basketball üìä

### Video Style & Approach

**Tone:** Analytical, thoughtful, data-driven  
**Pacing:** Steady, methodical, detail-oriented  
**Visuals:** Charts, graphs, statistics, diagrams, data visualizations  
**Language:** Technical, precise, analytical terminology

### Structure for "Studies Basketball" Videos

#### 1. Hook/Introduction (1-2 minutes)
**Goal:** Present an interesting data/statistical question

**Content:**
- Start with a basketball statistic or data point
- "Did you know that teams that maintain possession for X seconds score Y% more?"
- "Today we're going to learn how coding helps us analyze basketball data"
- Show a chart/graph that relates to the coding concept

**Example (Episode 1 - State Management):**
- Show possession statistics chart
- "This data tracks game states across 1000 possessions"
- "Let's learn how code manages this data"

#### 2. Core Content (10-15 minutes)
**Goal:** Teach coding concept through data analysis

**Approach:**
- Start with data/statistics
- Show how data is organized
- Connect data structure to coding concept
- Use charts, graphs, tables
- Analytical breakdowns
- Show the "why" and "how" behind the data

**Content Organization:**
- **Data/Statistics First** (3-5 min)
  - Present relevant basketball data
  - Show charts/graphs
  - Analyze the data
  - "What patterns do you see?"
  
- **Connect to Coding** (3-5 min)
  - "This is how code organizes this data"
  - Show data structures
  - Map statistics ‚Üí code structures
  - Show code examples with data
  
- **Analyze with Code** (3-5 min)
  - Show how code processes the data
  - Demonstrate calculations
  - Show data transformations
  - "Here's how code would analyze this"

#### 3. Interactive Checkpoint (1-2 minutes)
**Goal:** Check understanding with data analysis

**Content:**
- "Let's analyze this data together"
- Show a chart/graph
- Ask: "What would the code structure look like?"
- "How would you organize this data?"
- Pause for student response
- Reveal answer with explanation

#### 4. Application/Example (3-5 minutes)
**Goal:** Show real data analysis examples

**Content:**
- Show multiple data sets
- Demonstrate different analysis approaches
- "Here's how this works with different types of data"
- Show code processing data
- Connect to real-world analytics

#### 5. Career Connection (2-3 minutes)
**Goal:** Connect to data/analytics careers

**Content:**
- "Data scientists use these exact concepts"
- "Analysts organize data this way"
- Show how professionals use these concepts
- "Sports analytics uses state management"
- Show real data science examples
- Basketball connection: "Just like analyzing game data, data scientists analyze all kinds of data"

#### 6. Wrap-Up/Next Steps (1-2 minutes)
**Goal:** Reinforce learning, preview next steps

**Content:**
- Quick recap: "We learned [concept] through data analysis"
- "Try analyzing game statistics yourself!"
- Preview next episode
- Call to action: "Complete the exercises to practice"

### Recording Tips for "Studies Basketball" Style

**Energy:**
- Calm, thoughtful, analytical
- Sound like a teacher/analyst
- Clear, precise delivery
- Pause for thinking

**Visuals:**
- Charts, graphs, tables
- Data visualizations
- Code examples
- Diagrams showing structure
- Clean, organized layouts

**Language:**
- Technical, precise
- Data terminology
- Analytical language
- "Let's analyze," "observe," "calculate"
- Then connect to code terms

**Examples:**
- Statistics and data
- Analytical breakdowns
- Data structures
- Code examples
- Professional analytics

## Focus Area 3: Interested in Stories üìñ

### Video Style & Approach

**Tone:** Narrative, engaging, character-driven  
**Pacing:** Storytelling rhythm, builds tension, character moments  
**Visuals:** Character illustrations, story scenes, narrative visuals  
**Language:** Story language, character dialogue, plot elements

### Structure for "Interested in Stories" Videos

#### 1. Hook/Introduction (1-2 minutes)
**Goal:** Hook with story/character moment

**Content:**
- Start with a story moment or character scene
- "Nova looked at the court and saw something no one else could see..."
- "Today we're going on an adventure with Nova and the crew"
- Set up the story conflict/challenge
- Introduce the coding concept as part of the story

**Example (Episode 1 - State Management):**
- "Nova's team was losing. The Shadow Press Scouts were everywhere, creating chaos."
- "But Nova had a secret: she could see the game in states."
- "Let's join Nova's adventure and learn how states work!"

#### 2. Core Content (10-15 minutes)
**Goal:** Teach coding concept through story narrative

**Approach:**
- Tell the story first
- Characters learn the concept
- Show character moments
- Use dialogue and character interactions
- Story drives the learning
- Coding concept emerges from story

**Content Organization:**
- **Story Narrative** (5-7 min)
  - Tell the episode story
  - Character moments
  - Story conflict
  - Character learning moments
  - "Nova discovered..."
  
- **Concept Explanation** (3-5 min)
  - "What Nova learned is called [concept]"
  - Explain the coding concept
  - Connect story moment to concept
  - "Just like Nova, you can..."
  
- **Story Application** (3-5 min)
  - Continue the story
  - Show concept in action
  - Character success
  - "Nova used [concept] to..."
  - Story resolution

#### 3. Interactive Checkpoint (1-2 minutes)
**Goal:** Check understanding through story questions

**Content:**
- "What do you think Nova should do?"
- "How would you help the crew?"
- Story-based questions
- Pause for student response
- Continue story with explanation

#### 4. Application/Example (3-5 minutes)
**Goal:** Show story examples and character growth

**Content:**
- Show different story scenarios
- "What if the crew faced this challenge?"
- Character problem-solving
- Multiple story examples
- Character growth moments

#### 5. Career Connection (2-3 minutes)
**Goal:** Connect to careers through story

**Content:**
- "Nova's skills are just like a software engineer's"
- "Game developers create stories like this"
- Show how professionals use these concepts
- "Storytelling is part of coding too"
- Show real examples
- Story connection: "Just like Nova's adventure, coding is an adventure"

#### 6. Wrap-Up/Next Steps (1-2 minutes)
**Goal:** Reinforce learning, preview next story

**Content:**
- Quick recap: "We learned [concept] through Nova's story"
- "What will happen next in Nova's adventure?"
- Preview next episode story
- Call to action: "Complete the exercises to continue the story"

### Recording Tips for "Interested in Stories" Style

## Recording Specifications

### Technical Requirements (All Three Styles)

**Video Format:**
- Resolution: 1080p minimum (1920x1080)
- Aspect Ratio: 16:9
- Frame Rate: 30fps (or 60fps for action)
- Format: MP4 (H.264 codec)

**Audio:**
- Clear narration
- Background music (optional, style-appropriate)
- Sound effects (optional, for story style)
- No background noise

**Duration:**
- Target: 18-25 minutes per episode
- Can vary by style (Basketball: 20-25 min, Studies: 18-22 min, Stories: 20-27 min)

### Style-Specific Requirements

**Love Basketball:**
- Game footage integration
- Court diagrams
- Play breakdowns
- Dynamic editing
- Fast-paced visuals

**Studies Basketball:**
- Charts/graphs clearly visible
- Data visualizations
- Code examples readable
- Clean, organized layouts
- Pause for data analysis

**Stories:**
- Character illustrations
- Story visuals
- Narrative flow
- Character moments
- Story pacing

## Production Workflow

### Pre-Production (For Each Episode)

1. **Choose Primary Style** (or create all three)
2. **Gather Content:**
   - Basketball: Game footage, play diagrams
   - Studies: Data, charts, statistics
   - Stories: Story content, character visuals
3. **Script Writing:**
   - Follow structure for chosen style
   - Include Career Connections
   - Add interactive checkpoints
4. **Visual Preparation:**
   - Gather/create visuals
   - Prepare graphics
   - Source footage/images

### Production (Recording)

1. **Record Narration:**
   - Match energy to style
   - Clear, engaging delivery
   - Pause for interactive moments
2. **Edit Video:**
   - Add visuals
   - Sync with narration
   - Add transitions
   - Include interactive checkpoints
3. **Add Career Connections:**
   - Integrate career section
   - Style-appropriate examples
   - Real-world connections

### Post-Production

1. **Review & Refine:**
   - Check pacing
   - Verify content accuracy
   - Ensure engagement
2. **Add Metadata:**
   - Titles, descriptions
   - Timestamps for sections
   - Resource links
3. **Export & Upload:**
   - Final video file
   - Upload to hosting platform
   - Add to website
