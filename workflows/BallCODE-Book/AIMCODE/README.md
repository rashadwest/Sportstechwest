## AIMCODE Canonical Hub

This is the single source of truth for using AIMCODE efficiently in this repo.

- Complexity paths: Simple (Lightweight), Medium (Hybrid), Complex (Full)
- CLEAR everywhere (brief for Simple, full for Complex)
- Build–Measure–Learn embedded in every task
- Reuse then improve via exemplars and patterns

### Start Here
- Read: `../AIMCODE-Quick-Start-10min.md`
- Use: Issue template “AIMCODE Task” and PR template
- Reference: `../AIMCODE-RESEARCH-REFERENCE.md`

### Workflow Overview
1. Choose complexity (pre-task decision tree in the Issue)
2. Fill CLEAR (≤5 lines for Simple)
3. Add BML micro (Build, Measure, Learn)
4. Implement; link to relevant guides
5. Measure and record Learnings in PR
