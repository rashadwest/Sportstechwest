## ğŸ“Š WORKFLOW PHASES

### Phase 1: Initial Analysis
**Purpose:** Understand current state and requirements

**Steps:**
1. **Apply Prioritization Framework**
   - Identify task type
   - Select critical questions
   - Answer critical + quick questions

2. **Document Insights**
   - Record key findings
   - Identify gaps
   - Note pain points

3. **Create Analysis Document**
   - Document answers
   - Highlight insights
   - Note uncertainties

**Deliverable:** Initial analysis document with insights

**Time:** ~30-40% of total time

---

### Phase 2: Initial Design
**Purpose:** Design system features based on analysis

**Steps:**
1. **Translate Insights to Features**
   - Map insights to system features
   - Design solutions
   - Create implementation plan

2. **Apply 4 Principles**
   - Principle 3: Expert persona
   - Principle 1: Experiment
   - Principle 2: Human in loop
   - Principle 4: Always improve

3. **Create Design Document**
   - Document features
   - Explain design decisions
   - Note assumptions

**Deliverable:** Initial design document with features

**Time:** ~30-40% of total time

---

### Phase 3: Validation
**Purpose:** Check if design addresses analysis insights

**Steps:**
1. **Check Traceability**
   - Map design features to analysis questions
   - Verify all critical insights addressed
   - Identify missing connections

2. **Validate Completeness**
   - Check if all gaps addressed
   - Verify pain points solved
   - Confirm requirements met

3. **Identify Discrepancies**
   - Find analysis insights not in design
   - Find design features not in analysis
   - Note inconsistencies

**Deliverable:** Validation report with gaps and discrepancies

**Time:** ~10-15% of total time

---

### Phase 4: Refinement
**Purpose:** Refine both analysis and design based on validation

**Steps:**
1. **Refine Analysis**
   - Add missing questions if needed
   - Clarify uncertainties
   - Update insights based on design

2. **Refine Design**
   - Add missing features
   - Remove unnecessary features
   - Adjust based on analysis

3. **Update Traceability**
   - Update question-to-feature mapping
   - Document new connections
   - Remove invalid connections

**Deliverable:** Refined analysis and design documents

**Time:** ~10-15% of total time

---

### Phase 5: Iteration Decision
**Purpose:** Decide if another iteration is needed

**Decision Criteria:**
- **Convergence:** Analysis and design aligned?
- **Completeness:** All critical insights addressed?
- **Quality:** Design meets requirements?
- **Time:** More iterations feasible?

**Decision:**
- **Yes (Iterate):** Return to Phase 1 with refined understanding
- **No (Proceed):** Move to implementation

**Deliverable:** Decision and rationale

**Time:** ~5% of total time

## ğŸ¯ VALIDATION CHECKPOINTS

### Checkpoint 1: Analysis Completeness
**Questions:**
- Are all critical questions answered?
- Are insights clearly documented?
- Are gaps identified?

**Action if Failed:**
- Return to Phase 1
- Answer missing questions
- Document additional insights

---

### Checkpoint 2: Design Completeness
**Questions:**
- Are all insights addressed in design?
- Are features clearly defined?
- Is implementation plan clear?

**Action if Failed:**
- Return to Phase 2
- Add missing features
- Refine design

---

### Checkpoint 3: Traceability
**Questions:**
- Can all features be traced to questions?
- Are all questions addressed by features?
- Are connections clear?

**Action if Failed:**
- Return to Phase 4
- Update traceability matrix
- Document connections

---

### Checkpoint 4: Convergence
**Questions:**
- Are analysis and design aligned?
- Are discrepancies resolved?
- Is quality acceptable?

**Action if Failed:**
- Return to Phase 4
- Refine both analysis and design
- Iterate again

### When to Iterate:
- **High Uncertainty:** Many unknowns â†’ Iterate more
- **High Complexity:** Complex system â†’ Iterate more
- **High Stakes:** Critical feature â†’ Iterate more
- **Low Time:** Time pressure â†’ Iterate less

### Iteration Guidelines:
- **Simple Tasks:** 1 iteration (initial pass)
- **Medium Tasks:** 2 iterations (initial + refinement)
- **Complex Tasks:** 3+ iterations (as needed)

### Time Allocation:
- **Initial Pass:** 70% of time
- **Refinement Pass:** 20% of time
- **Final Pass:** 10% of time
