### Step 3: Write Act I (Setup and Problem)
**Target:** 400-500 words

**Checklist:**
- [ ] Start with basketball action (NOT concept explanation)
- [ ] Show the problem: Team/player struggling
- [ ] Introduce characters: Nova, Arc, Coach Circuit
- [ ] Show failed attempts
- [ ] Arc detects problem (use exact lines from outline)
- [ ] Coach gives wisdom (use exact lines from outline)
- [ ] End with Nova's realization: "Maybe I need to..."

**Key Rules:**
- ‚úÖ Basketball terminology used naturally
- ‚úÖ Show, don't tell (show frustration, don't just say "Nova was frustrated")
- ‚úÖ Let concepts emerge FROM basketball needs
- ‚ùå Don't explain AI/math/coding concepts first
- ‚ùå Don't start with "Today we're learning about..."

---

### Step 4: Write Act II (Learning and Struggle)
**Target:** 400-500 words

**Checklist:**
- [ ] Pressure intensifies (basketball challenge gets worse)
- [ ] Nova practices the move (Pound/Crossover/In & Out)
- [ ] Arc recognizes pattern (use exact lines from outline)
- [ ] Math emerges naturally (counting, time, sequences)
- [ ] Coding connection made (blocks, if/then, loops)
- [ ] Nova struggles/resists at first
- [ ] Coach responds with wisdom
- [ ] Nova realizes the connection

**Key Rules:**
- ‚úÖ Concepts emerge FROM basketball practice
- ‚úÖ Use exact character lines from outline
- ‚úÖ Show the struggle, not just the solution
- ‚úÖ Make it feel like discovery, not lesson

---

### Step 5: Write Act III (Resolution and Application)
**Target:** 400-500 words

**Checklist:**
- [ ] Final challenge (high stakes, time pressure)
- [ ] Nova applies the concept (uses the move/sequence)
- [ ] Success! (Team/player succeeds)
- [ ] Reflection (Nova understands the connection)
- [ ] Future setup (hint at next book/concept)

**Key Rules:**
- ‚úÖ Basketball success demonstrates mastery
- ‚úÖ Show the connection clearly
- ‚úÖ End with understanding and confidence
- ‚úÖ Keep it engaging and adventurous

---

### Step 6: Review and Polish
**Checklist:**
- [ ] Word count: 1,200-1,800 words total
- [ ] 3-act structure: All acts complete
- [ ] Basketball action starts the story
- [ ] Basketball terminology used naturally
- [ ] AI/math/coding concepts emerge FROM basketball
- [ ] Character lines match outline exactly
- [ ] Story feels engaging, not like a lesson
- [ ] No TODO comments or placeholders

**Quality Check:**
- [ ] Story starts with basketball action ‚úÖ
- [ ] Basketball terminology used naturally ‚úÖ
- [ ] AI/math/coding concepts emerge FROM basketball ‚úÖ
- [ ] 3-act structure followed ‚úÖ
- [ ] Basketball success demonstrates mastery ‚úÖ
- [ ] Engaging and adventurous (70% story, 30% skill) ‚úÖ

**AIMCODE Validation:**
- [ ] CLEAR framework applied (objectives, logic, examples, adaptation, results)
- [ ] Alpha Evolve layers complete (basketball ‚Üí concepts ‚Üí learning ‚Üí game)
- [ ] Expert principles applied (Zhang, Resnick, Reggio, Hassabis, Jobs)
- [ ] Game integration points identified
- [ ] Metrics setup planned

**For Full Checklist:** See `AIMCODE-STORY-CREATION-CHECKLIST.md`

# TASK 2: CREATING THE BOOKS (Formatting/Preparing)

**Time Estimate:** 15-20 minutes per book  
**Output:** Formatted book ready for recording  
**Format:** Clean, readable document with proper structure

---

## Step-by-Step Process

### Step 1: Open Your Draft
- [ ] Open: `Book-[Number]-[Title]-DRAFT.md`
- [ ] Create new file: `Book-[Number]-[Title]-FINAL.md`

---

### Step 2: Format the Document
**Checklist:**
- [ ] Add title at top: `# Book [Number]: [Title]`
- [ ] Add subtitle: `Dribble Level [Number] - [Concept]`
- [ ] Add author line (if needed)
- [ ] Add date line (if needed)
- [ ] Format Act I heading: `## Act I: [Title]`
- [ ] Format Act II heading: `## Act II: [Title]`
- [ ] Format Act III heading: `## Act III: [Title]`
- [ ] Ensure proper paragraph breaks
- [ ] Remove any placeholder text or TODOs
- [ ] Check spelling and grammar

**Example Format:**
```markdown
# Book 1: The Foundation Block
## Dribble Level 1 - Foundation Blocks

---

## Act I: The Press

[Story content here...]

---

## Act II: The Foundation

[Story content here...]

---

## Act III: Breaking Through

[Story content here...]
```

---

### Step 3: Add Page Breaks (If Needed)
- [ ] If creating PDF/print version, add page breaks
- [ ] If digital only, skip this step
- [ ] Ensure readability on screen

---

### Step 4: Final Review
**Checklist:**
- [ ] Document is clean and readable
- [ ] All formatting is consistent
- [ ] No errors or typos
- [ ] Ready for recording
- [ ] File saved as: `Book-[Number]-[Title]-FINAL.md`

---

### Step 5: Save and Mark Complete
- [ ] Save final formatted version
- [ ] Mark Task 2 complete for this book
- [ ] Move to Task 3

# TASK 3: RECORDING AND SENDING TO AI

**Time Estimate:** 20-30 minutes per book  
**Output:** Recorded video/audio of book reading  
**Format:** Video file ready for upload to website

---

## Step-by-Step Process

### Step 1: Prepare for Recording
**Checklist:**
- [ ] Open: `Book-[Number]-[Title]-FINAL.md`
- [ ] Set up recording space (quiet, good lighting)
- [ ] Test recording setup (browser-based recording)
- [ ] Have book text visible on screen
- [ ] Close distractions (phone, email, other tabs)
- [ ] Ensure good internet connection (for browser recording)

**Recording Setup:**
- [ ] Browser recording tool ready (e.g., Loom, Screencastify, browser built-in)
- [ ] Screen visible with book text
- [ ] Audio working (microphone test)
- [ ] Video quality set (720p minimum, 1080p preferred)

---

### Step 2: Record the Book Reading
**Checklist:**
- [ ] Start recording
- [ ] Read title and subtitle clearly
- [ ] Read Act I with expression and pacing
- [ ] Pause between acts (natural breaks)
- [ ] Read Act II with expression and pacing
- [ ] Pause between acts
- [ ] Read Act III with expression and pacing
- [ ] End recording

**Recording Tips:**
- ‚úÖ Speak clearly and at moderate pace
- ‚úÖ Use expression (make it engaging)
- ‚úÖ Pause for emphasis at key moments
- ‚úÖ Don't rush - quality over speed
- ‚úÖ If you make a mistake, pause and re-read that section

---

### Step 3: Review Recording
**Checklist:**
- [ ] Watch/listen to recording
- [ ] Check audio quality (clear, no background noise)
- [ ] Check video quality (readable text, good framing)
- [ ] Verify all content was read
- [ ] Note any issues or needed re-recordings

**If Issues Found:**
- [ ] Re-record specific sections if needed
- [ ] Or re-record entire book if major issues
- [ ] Don't proceed until quality is good

---

### Step 4: Save Recording
**Checklist:**
- [ ] Save file with clear name: `Book-[Number]-[Title]-Recording.[format]`
- [ ] Example: `Book-1-Pound-Dribble-Recording.mp4`
- [ ] Save to project folder: `Recordings/` or similar
- [ ] Note file location for upload

---

### Step 5: Send to AI for Processing
**Checklist:**
- [ ] Upload recording file to chat/workspace
- [ ] Provide context: "This is Book [Number]: [Title]"
- [ ] Request: "Please process this recording and prepare it for website upload"
- [ ] Include any specific instructions (e.g., "Add to paywall section")

**What AI Will Do:**
- [ ] Review recording quality
- [ ] Prepare file for website upload
- [ ] Create/update website page for book
- [ ] Set up paywall protection
- [ ] Add book to payment options

---

### Step 6: Mark Complete
- [ ] Mark Task 3 complete for this book
- [ ] Mark entire book complete
- [ ] Move to next book (repeat process)
