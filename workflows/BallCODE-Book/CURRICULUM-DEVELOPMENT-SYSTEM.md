## üìã CURRICULUM DEVELOPMENT WORKFLOW

### Phase 1: Concept Development
**Duration:** 1-2 weeks  
**Output:** Book proposal document

#### Step 1.1: Identify Learning Objective
- **What Python concept will this book teach?**
  - Examples: Sequences, Conditionals, Loops, Functions, Data Structures
- **What basketball context fits this concept?**
  - Examples: Breaking press (sequences), Reading defense (conditionals), Patterns (loops)
- **What grade level is this for?**
  - Grades 3-5: Concepts through blocks
  - Grades 6-8: Concepts + Python syntax
  - Grades 9-12: Full Python programming

#### Step 1.2: Create Book Proposal
**Template:** Use `BOOK-[N]-PROPOSAL-[CONCEPT].md` format

**Required Sections:**
1. **Title & Concept**
   - Book number and title
   - Python concept to teach
   - Basketball context

2. **Learning Objectives (Measurable)**
   - By end of book, students will be able to:
     - [Specific, measurable outcome 1]
     - [Specific, measurable outcome 2]
     - [Specific, measurable outcome 3]

3. **Basketball Story Framework**
   - Act I: Setup/problem (basketball situation)
   - Act II: Struggle/learning (concept emerges)
   - Act III: Resolution/success (concept mastery)

4. **Python Progression**
   - Phase 1: Block coding example
   - Phase 2: Transition bridge example
   - Phase 3: Python code example

5. **Standards Alignment**
   - CSTA K-12 Computer Science Standards
   - Common Core Mathematics
   - Next Generation Science Standards

6. **Game Integration**
   - Exercise name
   - Skills practice
   - Success criteria

**Deliverable:** Complete book proposal document

---

### Phase 2: Story Development
**Duration:** 2-4 weeks  
**Output:** Complete story narrative

#### Step 2.1: Story Outline
**Template:** Use `WRITING-OUTLINE-BOOK-[N]-SIMPLE.md` format

**Required Elements:**
- **3-Act Structure:**
  - Act I: Basketball problem/situation (70% story)
  - Act II: Learning/concept introduction (20% story, 10% skill)
  - Act III: Success/resolution (70% story, 30% skill)

- **Basketball-First Rule:**
  - ‚úÖ Start with basketball action
  - ‚úÖ Use basketball terminology naturally
  - ‚úÖ Show concepts solving basketball problems
  - ‚ùå Don't start with "Today we're learning X"
  - ‚ùå Don't explain concepts before basketball context

#### Step 2.2: Story Writing
**Template:** Use `WRITING-GUIDE-BOOK-[N]-PICTURE-BOOK.md` format

**Story Requirements:**
- **Target Length:** 1,000-1,800 words (12-page storybook)
- **Target Age:** Grades 3-8 (ages 8-14)
- **Tone:** Adventurous, engaging, kid-friendly
- **Character Consistency:** Follow Character & World Bible

**Code Integration:**
- Code appears AFTER basketball situation is established
- Code shown visually, not explained abstractly
- Code solves basketball problems in real-time

**Deliverable:** Complete story narrative

---

### Phase 3: Production Outline Creation
**Duration:** 1 week  
**Output:** Master storybook production outline

#### Step 3.1: Create Master Outline
**Template:** Use `Book-[N]-Master-Storybook-Production-Outline.md` format

**Required Sections:**
1. **Quick Start Guide**
   - How to use document
   - Code copy/paste instructions
   - Quick reference

2. **Code Integration Guide**
   - Target audience specs
   - Character appearance guidelines
   - 5 code visual presentation options
   - Copy/paste instructions
   - Image generation prompts
   - 12-page storybook format
   - Code integration checklist

3. **Chapter Structure (7 chapters)**
   Each chapter includes:
   - Story text
   - Python code block
   - Curriculum learning note
   - Image prompt with exact code
   - Page creation instructions

4. **End Credits**
   - Learning summary
   - Next tutorial message
   - Image prompt

5. **Teacher Resources**
   - Learning objectives
   - Standards alignment
   - Assessment framework

**Deliverable:** Complete master production outline

---

### Phase 4: Visual Asset Development
**Duration:** 2-3 weeks  
**Output:** All visual assets

#### Step 4.1: Image Generation
**Using Master Production Outline:**

1. **For Each Chapter:**
   - Extract image prompt from outline
   - Include exact code from "CODE TO DISPLAY IN IMAGE (EXACT)" section
   - Generate image using AI image generator
   - Verify code is readable in image

2. **Image Requirements:**
   - Code clearly visible and readable
   - Monospace font style
   - Proper contrast (light code on dark, or dark code on light)
   - Age-appropriate (grades 3-8)
   - Character consistency

3. **Image Styles:**
   - Option A: Floating Hologram
   - Option B: Thought Bubble
   - Option C: AR Overlay
   - Option D: Game Command Display
   - Option E: Robot Interface

**Deliverable:** 12 images (1 per page, or 6 two-page spreads)

---

### Phase 5: Book Assembly
**Duration:** 1-2 weeks  
**Output:** Complete storybook

#### Step 5.1: Page Layout
**Using Master Production Outline:**

1. **For Each Page:**
   - Follow page creation instructions
   - Place story text as specified
   - Integrate code using specified visual style
   - Place illustration as specified

2. **Layout Options:**
   - Two-page spread (left: text, right: illustration with code)
   - Single page (top: illustration with code, bottom: text)
   - Code callout (full illustration, code in callout box)

#### Step 5.2: Quality Check
**Use Code Integration Checklist:**
- [ ] Code copied EXACTLY from document
- [ ] Code readable and clearly visible
- [ ] Code uses specified visual presentation style
- [ ] Code placement doesn't interfere with text
- [ ] Code appears on same page as story text
- [ ] Code formatting matches document
- [ ] Code age-appropriate
- [ ] Code integrates naturally with illustration

**Deliverable:** Complete 12-page storybook

---

### Phase 6: Teacher Resources
**Duration:** 1 week  
**Output:** Complete teacher guide

#### Step 6.1: Create Teacher Guide
**Required Sections:**

1. **Overview**
   - Book summary
   - Learning objectives
   - Grade level recommendations
   - Duration estimate

2. **Standards Alignment**
   - CSTA K-12 Computer Science Standards
   - Common Core Mathematics
   - Next Generation Science Standards
   - 21st Century Skills

3. **Lesson Plans**
   - Week-by-week breakdown
   - Daily activities
   - Differentiation strategies
   - Extension activities

4. **Assessment Tools**
   - Formative assessments
   - Summative assessments
   - Rubrics
   - Answer keys

5. **Resources**
   - Student handouts
   - Visual aids
   - Reference materials
   - Supplementary resources

**Deliverable:** Complete teacher guide

---

### Phase 7: Game Integration
**Duration:** 2-3 weeks  
**Output:** Game exercise

#### Step 7.1: Design Game Exercise
**Based on Book Content:**

1. **Exercise Name**
   - Connect to book story
   - Clear skill focus

2. **Skills Practice**
   - Block coding practice
   - Python code practice
   - Concept application

3. **Success Criteria**
   - Clear completion requirements
   - Measurable outcomes

4. **Integration Points**
   - QR code in book
   - Short URL access
   - No login required (first play)

**Deliverable:** Game exercise integrated with book

## üéì GRADE-LEVEL PROGRESSION

### Grades 3-5 (Elementary)
**Focus:** Concepts through blocks and stories

**Books 1-3:**
- Block coding: Drag blocks to create programs
- Python exposure: See Python code, understand concepts
- Not required to write Python yet

**Assessment:**
- Block coding exercises (70%)
- Story comprehension (20%)
- Concept understanding (10%)

---

### Grades 6-8 (Middle School)
**Focus:** Concepts + Python syntax introduction

**Books 1-3:**
- Block coding: Create programs
- Python: Write code independently
- Transition: See blocks ‚Üí code connection

**Assessment:**
- Block coding exercises (30%)
- Python code writing (50%)
- Project-based assessments (20%)

---

### Grades 9-12 (High School)
**Focus:** Full Python programming

**All Books:**
- Full Python implementation
- Advanced concepts: Functions, classes, data structures
- Real-world applications

**Assessment:**
- Complex Python programs (40%)
- Project portfolios (40%)
- Real-world problem solving (20%)

## üìä ASSESSMENT FRAMEWORK

### Formative Assessment (During Learning)
**Purpose:** Check understanding during learning process

**Methods:**
- Story comprehension questions
- Concept understanding checks
- Block coding exercises
- Quick concept quizzes
- Code reading exercises

**Frequency:** Weekly during each book unit

---

### Summative Assessment (After Learning)
**Purpose:** Measure mastery after learning

**Methods:**
- Game exercise completion
- Python code writing assessments
- Project-based assessments
- Portfolio development

**Frequency:** End of each book unit

---

### Assessment Rubrics

#### Novice Level
- Can identify concepts in stories
- Needs guidance to create blocks
- Can read simple Python code with help
- Cannot write Python code independently

#### Developing Level
- Can create simple block programs
- Can read Python code with minimal help
- Can write simple Python code with guidance
- Understands basic concepts

#### Proficient Level
- Can create block programs independently
- Can read Python code independently
- Can write Python code independently
- Can apply concepts to solve problems

#### Advanced Level
- Can create complex block programs
- Can read and understand complex Python code
- Can write complex Python code
- Can create original solutions using concepts

## üîó THREE-PHASE LEARNING PATHWAY

### Phase 1: Sports Language (Block Coding)
**Duration:** Weeks 1-2 of each book
**Focus:** Learn concepts through basketball terminology

**What Students Do:**
- Drag blocks together using basketball terms
- Create programs visually (no typing required)
- See immediate feedback in game exercises
- Understand concepts before syntax

---

### Phase 2: Transition Bridge
**Duration:** Weeks 3-4 of each book
**Focus:** Connect blocks to code

**What Students Do:**
- See blocks and code side-by-side
- Understand that blocks = code
- Recognize Python syntax patterns
- Build confidence for Phase 3

---

### Phase 3: Python Learning
**Duration:** Weeks 5-8 of each book
**Focus:** Write real Python code

**What Students Do:**
- Write Python code independently
- Apply concepts in real code
- Debug and test programs
- Create original solutions
