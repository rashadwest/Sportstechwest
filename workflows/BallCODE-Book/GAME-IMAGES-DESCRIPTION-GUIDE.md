# Quick Guide: Describing Game Images

**Copyright ¬© 2025 Rashad West. All Rights Reserved.**

**Purpose:** Simple guide to describe game screenshots for documentation  
**Time:** ~5-10 minutes per image

---

## üéØ What You Need to Describe

For each game image, answer these questions:

### 1. **What Level Is This?**
- Level name/number (e.g., "Watson1 Level 3", "Foundation Block Tutorial")
- Difficulty level (Tutorial/Intermediate/Advanced)

### 2. **What's the Objective?**
- What is the player trying to do? (e.g., "Move to the basket", "Dribble past defenders")
- What makes this level unique?

### 3. **What Do You See?**
- **Court layout:** Basketball court, hoops, lines, obstacles
- **Player position:** Where is the player/ball?
- **UI elements:** Buttons, menus, score, timer
- **Block coding area:** What blocks are visible? What's arranged?
- **Game state:** Is it running? Paused? Success screen? Failure screen?

### 4. **What Coding Concept?**
- What coding skill is being taught? (e.g., "If/Then statements", "Loops", "Variables")
- How does the basketball context relate to the coding concept?

```
[Any important mechanics or features]
```

## üöÄ Quick Workflow

1. **Open the image** (in Finder or image viewer)
2. **Fill in the template** above (5-10 minutes)
3. **Save descriptions** in a document (e.g., `Game-Image-Descriptions.md`)
4. **Repeat for each image**

## üí° Tips for Low-Focus Work

**Since you're in a low-focus window:**

1. **Do 2-3 images at a time** - Don't try to do all at once
2. **Focus on basics** - Level name, objective, what you see
3. **Skip details** - You can add more detail later
4. **Use voice notes** - If typing is too much, record voice descriptions
5. **Set a timer** - 10 minutes per image, then take a break
