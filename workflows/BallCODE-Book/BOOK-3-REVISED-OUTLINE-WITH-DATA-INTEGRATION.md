## üéØ WHAT IS BALL DON'T LIE?

**Ball Don't Lie** is a **FREE basketball statistics API** that provides:
- Real NBA player statistics (points, rebounds, assists, etc.)
- Game data (scores, schedules, results)
- Team information
- Historical data

**Think of it like:** A library of basketball statistics that you can "ask questions" to using code. Instead of looking up stats manually, you write code that fetches the data automatically.

**Example:** 
- "Show me LeBron James's points per game"
- "What's the Lakers' win record?"
- "Which player scored the most in last night's game?"

**For Kids:** It's like having a robot assistant that knows all basketball statistics and can answer questions instantly!

## üíª PYTHON CURRICULUM CONNECTION

### Python Concept 1: Loops and Repetition (Primary)

**Block Coding (Phase 1):**
```
REPEAT [fake left] 3 TIMES
THEN [go right]
```

**Transition Bridge (Phase 2):**
```
Block: REPEAT [fake left] 3 TIMES
Code:  for i in range(3): fake_left()
```

**Python Syntax (Phase 3):**
```python
# In & Out Dribble - Loops
for i in range(3):
    fake_left()
go_right()
```

### Python Concept 2: Data Fetching (NEW - Simplified)

**Block Coding (Phase 1):**
```
FETCH [opponent data] FROM [Ball Don't Lie API]
```

**Transition Bridge (Phase 2):**
```
Block: FETCH [opponent data]
Code:  data = fetch_data("opponent_stats")
```

**Python Syntax (Phase 3 - Simplified):**
```python
# Fetching data from API (simplified for kids)
data = fetch_opponent_stats()  # Simplified function
# Or: data = get_stats("opponent")  # Even simpler
```

**Note:** For younger grades (3-5), we show the concept without actual API code. For older grades (6-8), we can show simplified API calls.

### Python Concept 3: Looping Through Data (NEW - Simplified)

**Block Coding (Phase 1):**
```
LOOP THROUGH [game data]
CHECK [reaction pattern]
```

**Transition Bridge (Phase 2):**
```
Block: LOOP THROUGH [game data]
Code:  for game in data: check_pattern(game)
```

**Python Syntax (Phase 3 - Simplified):**
```python
# Looping through data to find patterns
for game in opponent_data:
    if game.reaction == "left":
        left_count += 1
    else:
        right_count += 1

# Find pattern: 87% left, 13% right
```
