## üìê STORY STRUCTURE (3 Acts)

### **ACT I: The Lost Score (~450 words)**

#### **Opening Scene:**
- Game situation: Score is close, need to track it
- **Problem:** Score keeps getting lost/forgotten
- Nova: "How do we remember the score?"
- **Stakes:** Can't make decisions without knowing score

#### **Character Introduction:**
- **Nova:** Frustrated, can't track score
- **Arc:** "Data storage needed. Variable system required."
- **Coach Circuit:** "Variables store data. Like a notebook that remembers."

#### **Inciting Incident:**
- Score changes, but no one remembers what it was
- Bad decision made because score was wrong
- Arc: "Variable system needed. Store data, retrieve data."
- Coach: "Learn to track data. Variables are your memory."

---

### **ACT II: Learning Variable Mastery (~550 words)**

#### **Rising Action:**
- Multiple things to track (score, time, fouls, stats)
- Nova tries to remember everything - fails
- **Struggle:** How do variables store data?

#### **Learning Moments:**

**1. Simple Variables (Scaffolding):**
- Score = 0, then Score = 2, then Score = 5
- Time = 10:00, then Time = 9:45, then Time = 9:30
- **Visual:** See variable change over time
- **Scaffolding:** Start with one variable, add more

**2. Complex Variables:**
- Player stats (points, assists, rebounds together)
- Game state (score, time, fouls together)
- Arrays and objects (collections of data)
- **Scaffolding:** Build complexity gradually

**3. Data Persistence:**
- Variables save data between plays
- Game progress tracked across sessions
- Statistics accumulate over time
- **Connection:** Variables remember, even when game stops

#### **Basketball Variables:**
- **Player Stats:** Points, assists, rebounds
- **Game Variables:** Score, time, fouls
- **Team Variables:** Wins, losses, ranking
- **Performance:** Shooting percentage, turnover rate

#### **Struggle:**
- Nova: "Too many things to track! How do I remember everything?"
- Coach: "Use variables. They're like containers that hold data."
- Arc: "Variable system: SET [variable] TO [value], GET [variable]."

#### **Turning Point:**
- Nova understands: "Variables store data. I can SET and GET values."
- Connection: "Like a scoreboard - it remembers the score for us."

---

### **ACT III: Variable-Driven Success (~500 words)**

#### **Climax:**
- Final seconds, need to know exact score
- Multiple variables to check (score, time, fouls)
- **Challenge:** Use variables to make the right decision

#### **Variable-Driven Solution:**
- Arc: "Current variables: Score = 45, Time = 0:03, Fouls = 4."
- Nova: "So I know we're down by 2, have 3 seconds, and they're in foul trouble?"
- Coach: "Yes. Variables give you information to make decisions."

#### **Basketball Success:**
- Nova checks variables: Score (down by 2), Time (3 seconds), Fouls (they have 4)
- Decision: Shoot three-pointer (need 3 to win, have time)
- Makes shot, wins game
- **Success:** Variable-driven decision works!

#### **Resolution:**
- Nova reflects: "Variables store data. They help me make decisions."
- Connection: "Coding uses variables too - store data, use it later."
- Future: "Now I can track anything in the game!"

## üéÆ GAME INTEGRATION

### **Phase 1: Block Coding**
```
SET [score] TO 0
IF [basket made] THEN INCREMENT [score]
DISPLAY [score]
GET [time remaining]
```

### **Phase 2: Transition Bridge**
```
Block: SET [score] TO 0
Code:  score = 0

Block: INCREMENT [score]
Code:  score += 1
```

### **Phase 3: Python Learning**
```python
# Variables track game state
score = 0
time_remaining = 600  # 10 minutes in seconds
fouls = 0

# Update variables
if basket_made():
    score += 2
    print(f"Score: {score}")

# Use variables in decisions
if time_remaining < 60 and score < opponent_score:
    shoot_three_pointer()
```
