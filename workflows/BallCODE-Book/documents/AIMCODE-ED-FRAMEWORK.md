### **S - Specify the Role**
**Define the AI's educational persona and expertise level.**

**Examples:**
- "Act as an experienced K-5 computer science educator teaching Python through basketball stories"
- "Act as a curriculum specialist for grades 6-8 with expertise in CSTA standards alignment"
- "Act as an assessment designer specializing in project-based learning for diverse learners"

- Include AIMCODE advisory board expertise (Zhang, Resnick, Reggio, Hassabis, Jobs)
- Specify grade level and subject area
- Include basketball framework context when relevant

---

### **M - Mention the Purpose**
**State the specific educational goal or deliverable.**

**Examples:**
- "Create a lesson plan for Book 1: The Foundation Block, focusing on sequences"
- "Develop an assessment rubric for conditional logic (Book 2)"
- "Write a teacher guide for integrating Book 3 with game exercises"
- "Design differentiation strategies for mixed-ability classrooms"

**BallCODE Enhancement:**
- Reference specific books, episodes, or game exercises
- Connect to curriculum pathway (Phase 1: Blocks ‚Üí Phase 2: Bridge ‚Üí Phase 3: Python)
- Link to learning objectives and standards

---

### **A - Define the Audience**
**Identify the learners and their characteristics.**

**Examples:**
- "Grades 3-5 students (ages 8-11), mixed ability levels, some with no coding experience"
- "Grades 6-8 students (ages 11-14), varying skill levels, including English language learners"
- "First-year computer science students, diverse backgrounds"
- "Postgraduate educators learning to integrate coding into curriculum"

**BallCODE Enhancement:**
- Specify grade level (3-5, 6-8, 9-12)
- Include ability level diversity
- Note special considerations (ELL, special needs, gifted)
- Reference multiple entry points (Reggio Emilia principle)

---

### **R - Pedagogical Constraints**
**Set educational parameters and requirements.**

**Standard Constraints:**
- **Bloom's Taxonomy Levels:** Remember, Understand, Apply, Analyze, Evaluate, Create
- **Standards Alignment:** CSTA, Common Core Math, NGSS, ELA standards
- **Course Outcomes (CO):** Specific learning objectives
- **Real-World Examples:** Basketball scenarios, data analysis, career connections

**BallCODE-Specific Enhancements:**

1. **Multiple Entry Points (Reggio Emilia):**
   - Story pathway (narrative learners)
   - Game pathway (kinesthetic learners)
   - Code pathway (logical learners)
   - Visual pathway (visual learners)

2. **Alpha Evolve Layers (Hassabis - Systematic Deep Learning):**
   - **Layer 1 (Foundation):** Basketball context, basic concepts
   - **Layer 2 (Application):** Game exercises, hands-on practice
   - **Layer 3 (Integration):** Python code connection, real-world application
   - **Layer 4 (Mastery):** Advanced applications, career connections

3. **Basketball Framework Integration (Zhang - Story-First):**
   - 70% story, 30% skill instruction
   - Concepts emerge from basketball needs
   - Basketball success = proof of learning

4. **Constructionist Activities (Resnick):**
   - Students build to learn
   - Hands-on game exercises
   - Project-based learning

5. **Simplicity & Design (Jobs):**
   - Clear, intuitive structure
   - "It just works" - no unnecessary complexity
   - Beautiful, engaging presentation

**Example R Section:**
```
R - Pedagogical Constraints:
- Bloom's levels: Remember, Understand, Apply
- CSTA alignment: 1B-AP-10, 1B-AP-11, 2-AP-12
- Common Core Math: 6.EE.A.2, MP.2, MP.7
- Multiple entry points: Story, game, code, visual pathways
- Alpha Evolve layers: Layer 1 (Foundation) ‚Üí Layer 2 (Application)
- Basketball framework: 70% story, 30% skill, concepts emerge from narrative
- Real-world examples: Breaking the press, sequences of moves
- Differentiation: Scaffolding for struggling learners, extensions for advanced
```

### **PED - Prompt ‚Üí Evaluate ‚Üí Develop**
**Iterative refinement cycle for continuous improvement.**

**Process:**
1. **Prompt:** Create initial prompt with S-M-A-R-T elements
2. **Evaluate:** Review output for:
   - Pedagogical alignment (Bloom's, standards)
   - Age-appropriateness
   - Basketball framework integration
   - Multiple entry points
   - Clear structure
3. **Develop:** Refine based on evaluation:
   - Simplify language (Jobs)
   - Add activities (Resnick - constructionist)
   - Improve engagement (Zhang - story-first)
   - Enhance differentiation (Reggio - multiple entry points)
   - Strengthen systematic progression (Hassabis - Alpha Evolve)

**Example PED Cycle:**
```
Initial Prompt: [S-M-A-R-T prompt]
‚Üì

- ‚úÖ Standards aligned
- ‚ö†Ô∏è Needs more basketball context
- ‚ö†Ô∏è Missing differentiation strategies
- ‚úÖ Age-appropriate
‚Üì

- Add basketball scenario examples
- Include scaffolding for struggling learners
- Add extension activities for advanced learners
- Enhance story-first approach
```

### Example 1: Creating a Lesson Plan for Book 1

```
S - Specify the Role:
Act as an experienced K-5 computer science educator teaching Python through basketball stories, with expertise in CSTA standards alignment and constructionist learning (Resnick).

M - Mention the Purpose:
Create a lesson plan for Book 1: The Foundation Block, Episode 1, focusing on sequences and basic programming concepts, aligned with CSTA 1B-AP-10 and Common Core Math MP.2.

A - Define the Audience:
Grades 3-5 students (ages 8-11), mixed ability levels, some with no coding experience, including English language learners and students with varying learning styles.

R - Pedagogical Constraints:
- Bloom's levels: Remember, Understand, Apply
- CSTA alignment: 1B-AP-10 (sequences), 1B-AP-11 (decompose problems)
- Common Core Math: MP.2 (reasoning), MP.7 (structure)
- Multiple entry points: Story (narrative), Game (kinesthetic), Code (logical), Visual (blocks)
- Alpha Evolve layers: Layer 1 (Foundation - basketball context) ‚Üí Layer 2 (Application - game exercise)
- Basketball framework: 70% story (breaking the press), 30% skill (sequence concepts)
- Real-world examples: Breaking the press sequences, basketball move sequences
- Differentiation: Scaffolding for struggling learners (simplified blocks), extensions for advanced (Python code preview)
- Constructionist: Students build sequences in game exercises

T - Output Format:
Lesson plan with: Hook (basketball story - 10 min), Direct Instruction (sequence concept - 15 min), Guided Practice (game exercise - 20 min), Independent Practice (block sequences - 15 min), Closure (connection to basketball - 5 min), Assessment (observation rubric), Differentiation (scaffolding/extensions), Materials (Book 1, game access, blocks).

PED - Prompt ‚Üí Evaluate ‚Üí Develop:
[Create lesson plan] ‚Üí [Evaluate: Check basketball context, age-appropriateness, standards alignment] ‚Üí [Develop: Add more basketball examples, simplify language, enhance game exercise connection]
```
