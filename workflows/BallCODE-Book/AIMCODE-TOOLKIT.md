## Quick Decision Tree

```
Is this educational content creation?
├─ YES → Use Full AIMCODE
└─ NO → Is this a pedagogical decision?
    ├─ YES → Use Full AIMCODE
    └─ NO → Is this content planning?
        ├─ YES → Use Lightweight AIMCODE
        └─ NO → Skip AIMCODE (use standard workflow)
```

## When to Use AIMCODE

### ✅ Use Full AIMCODE For:

**Educational Content Creation**
- Story/episode writing
- Book development
- Learning activity design
- Curriculum planning
- Exercise creation

**Pedagogical Decisions**
- Concept sequencing
- Learning objective design
- Assessment creation
- Teaching methodology
- Educational framework design

**Framework Development**
- Methodology changes
- Process improvements
- Quality standards
- Best practices documentation

### ⚡ Use Lightweight AIMCODE For:

**Content Planning**
- Outlines and structure
- Content strategy
- Project planning
- Story beats/plot structure

**Design Decisions**
- User experience choices
- Interface design
- Content organization
- Visual design decisions

### ❌ Skip AIMCODE For:

**Technical Tasks**
- Code implementation
- File formatting
- System configuration
- Database setup
- API integration

**Administrative Tasks**
- File organization
- Data entry
- Quick edits (typos, formatting)
- File naming/renaming
- Git operations

**Routine Tasks**
- Documentation updates
- Status updates
- Meeting notes
- Quick fixes

## Project Type Guide

### Type 1: Educational Content Creation
**Use:** Full AIMCODE  
**Examples:** Stories, episodes, books, learning activities

**Workflow:**
1. CLEAR (Full) - Define content objectives
2. Alpha Evolve - Build concept layers
3. Research (Tier 2) - Educational research
4. Experts - Zhang (primary), Resnick, Reggio, Jobs

**Time Estimate:** 30-45 minutes per story

### Type 2: Pedagogical Decisions  
**Use:** Full AIMCODE  
**Examples:** Concept sequencing, learning objectives, assessments

**Workflow:**
1. CLEAR (Full) - Define pedagogical objectives
2. Alpha Evolve - Build learning layers
3. Research (Tier 3) - Deep educational research
4. Experts - All AIMCODE advisors

**Time Estimate:** 45-60 minutes

### Type 3: Content Planning
**Use:** Lightweight AIMCODE  
**Examples:** Outlines, structure, content strategy

**Workflow:**
1. CLEAR (Standard) - Define plan objectives
2. Experts - Zhang, Jobs (design focus)
3. Skip: Alpha Evolve, Research (unless needed)

**Time Estimate:** 10-15 minutes

### Type 4: Technical Implementation
**Use:** Skip AIMCODE  
**Examples:** Code, formatting, system configuration

**Workflow:**
- Standard development workflow
- Follow project coding standards
- Use technical best practices

## AIMCODE Shortcuts

### Shortcut 1: "AIMCODE Lite"
**For:** Content planning, outlines, quick iterations  
**Process:** 
- CLEAR (Standard: C, L, R)
- Experts (2-3 relevant only)
- Done

**Time:** 5-10 minutes

### Shortcut 2: "AIMCODE Quick"
**For:** Iterations on existing content  
**Process:**
- CLEAR (Quick: C, R)
- Experts (relevant only)
- Done

**Time:** 2-5 minutes

### Shortcut 3: "AIMCODE Deep"
**For:** New methodology, major decisions, research papers  
**Process:**
- Full workflow
- Extended research (Tier 3)
- All experts consulted
- Detailed documentation

**Time:** 30+ minutes

## Build-Measure-Learn Integration

**AIMCODE is enhanced by the BML feedback loop:**

### Story Creation with BML
- **BUILD**: Apply AIMCODE to create story
- **MEASURE**: Track story effectiveness metrics
- **LEARN**: Refine story creation process
- **Repeat**: Improved process for next story

### Game Development with BML
- **BUILD**: Apply AIMCODE to create features
- **MEASURE**: Track feature performance metrics
- **LEARN**: Refine feature development process
- **Repeat**: Improved process for next feature

## Quality Checkpoints

After each AIMCODE phase, ask:

- **CLEAR:** Are objectives clear? Can we proceed?
- **Alpha Evolve:** Are layers solid? Ready for next?
- **Research:** Is evidence sufficient? Need more?
- **Experts:** Are principles applied? Quality met?
