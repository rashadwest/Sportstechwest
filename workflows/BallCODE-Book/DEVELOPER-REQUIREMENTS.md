# Developer Requirements - BallCODE Website Updates

**Goal:** Clear instructions for what you need from a developer to enable website updates  
**Website:** ballcode.co  
**Updates Needed:** Episode 1 page, contact info, navigation, sign-up button

---

## Important Note: I Don't Need Direct Access

**Clarification:** I (the AI assistant) don't need direct login access to your website. Instead, I can:
- Provide code and files for the developer to implement
- Give step-by-step instructions
- Review code the developer shares
- Troubleshoot issues

**What We Need:** Information about your website setup so I can provide the right code/instructions.

---

## Information Needed from Developer

### 1. Website Platform/Technology

**Question:** What platform is ballcode.co built on?

**Options:**
- [ ] WordPress
- [ ] Static HTML/CSS/JavaScript
- [ ] React/Next.js
- [ ] Wix/Squarespace
- [ ] Netlify/Vercel
- [ ] Other CMS (specify: ___________)

**Why:** Determines what code format and instructions I provide.

---

### 2. Hosting & Deployment

**Questions:**
- Where is the site hosted? (e.g., Netlify, Vercel, WordPress hosting, custom server)
- How is code deployed? (Git push, FTP, WordPress admin, etc.)
- Do you have Git repository? (If yes, what's the URL?)

**Why:** Determines deployment method and file structure.

---

### 3. Current Site Structure

**Questions:**
- What's the current file structure?
- Where are pages stored? (e.g., `/pages/`, root directory, database)
- How is navigation currently implemented?
- What's the current URL structure? (e.g., `ballcode.co/page-name`)

**Why:** Helps me provide code that fits your existing structure.

---

### 4. Access & Permissions

**Questions:**
- Can the developer add new pages?
- Can the developer edit existing pages?
- Can the developer modify navigation/menus?
- Can the developer upload files?
- What level of access does developer have? (Admin, Editor, etc.)

**Why:** Determines what changes are possible.

---

### 5. Branding & Styling

**Questions:**
- What are your brand colors? (Hex codes)
- What fonts are used?
- Do you have a logo file?
- What's the current design system/style guide?

**Why:** So I can match your existing design.

## Recommended Approach

### Option A: Developer Implements (Recommended)

**Workflow:**
1. **You provide me:** Website platform info, branding, requirements
2. **I provide:** Complete code files, instructions, specifications
3. **Developer:** Implements code, deploys, tests
4. **You/Developer:** Share any issues
5. **I provide:** Fixes and adjustments

**Advantages:**
- No security concerns (no direct access)
- Developer maintains control
- I provide guidance and code
- Faster implementation

---

### Option B: Developer Shares Code for Review

**Workflow:**
1. **Developer:** Shares current code/structure
2. **I review:** Provide specific changes needed
3. **Developer:** Implements changes
4. **Repeat:** Until complete

**Advantages:**
- I can see existing structure
- More targeted recommendations
- Better integration with existing code
