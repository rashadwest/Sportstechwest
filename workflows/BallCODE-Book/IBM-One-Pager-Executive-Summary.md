## How BallCODE Works

### Our Story-First Approach (70% Story, 30% Instruction)
We believe kids learn best when they're engaged, so each lesson is a basketball adventure that naturally teaches coding concepts. Students learn alongside relatable characters facing real challengesâ€”no prior coding knowledge required. We start with sports language they already know, then progressively introduce coding concepts.

**Example:** Instead of teaching "if/then" logic abstractly, we tell a story about a player reading a defense and making a decision. The coding concept emerges naturally from the basketball context.

### Our Three-Phase Learning Pathway

We've designed a progressive pathway that takes students from sports language to real Python programming:

**Phase 1: Sports Language (Block Coding)**
Students learn coding using basketball terminology in a visual block interfaceâ€”no typing required. They drag blocks together to create programs, getting immediate feedback through integrated game exercises.

**Phase 2: Transition Bridge**
Students see how sports language maps to code concepts. We show blocks and code side-by-side, helping them understand that blocks = code.

**Phase 3: Python Learning**
Students progress to Python syntax, learning the same concepts in a different representation. They end up with real-world programming skills.

## What We've Built: 12-Episode Curriculum

We've structured BallCODE as 12 self-contained episodes, each a complete learning unit that can be used independently or as part of a series. Here's what we're teaching:

1. **State Management** - Managing game states and transitions
2. **Conditional Logic** - If/then decision-making in basketball
3. **Loops & Repetition** - Defensive rotations and patterns
4. **Pattern Recognition** - Identifying zone defenses
5. **Data Modeling** - Tracking possessions and game events
6. **Functions** - Reusable plays and sets
7. **Arrays & Collections** - Managing rosters and lineups
8. **Probability & Math** - Shot selection and expected value
9. **Algorithms** - Strategy and optimization
10. **Debugging** - Finding and fixing errors
11. **Data Schemas** - Building better models
12. **AI Integration** - Using AI tools effectively

### Each Episode Structure:
Every episode follows the same proven format:
- **Story Narrative** (1,000-1,800 words) - An engaging basketball adventure that introduces the concept
- **Skill Pit-Stop** - Direct instruction on the coding concept (the "teaching moment")
- **Try-It Exercise** - Hands-on practice in our integrated game (students actually code)
- **Mastery Lab** - Extended practice and assessment for deeper learning
