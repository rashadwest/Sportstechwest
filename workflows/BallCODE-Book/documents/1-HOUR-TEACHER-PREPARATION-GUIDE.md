## ğŸ® PART 2: TRY IT YOURSELF (20 minutes)

### Book 1: Sequences (7 minutes)

**Your Task:** Create a sequence to break the press

**Steps:**
1. Open BallCODE game
2. Drag `START` block to code area
3. Drag `MOVE FORWARD` below START
4. Drag `PASS BALL` below MOVE FORWARD
5. Drag `MOVE TO POSITION` below PASS BALL
6. Click RUN
7. Watch player execute the sequence

**What You Learned:**
- Blocks execute top to bottom
- Order matters (can't pass before you move)
- Visual feedback shows it working

**Try This:**
- Change the order - what happens?
- Add more blocks - make it longer
- Remove a block - what breaks?

### Book 2: Conditionals (7 minutes)

**Your Task:** Create a conditional for one-on-one

**Steps:**
1. Drag `START` block
2. Drag `MOVE FORWARD` below START
3. Drag `IF DEFENDER NEAR` below MOVE FORWARD
4. Drag `THEN CROSSOVER` inside the IF block
5. Drag `ELSE DRIVE` in the else branch
6. Click RUN
7. Watch player react to defender

**What You Learned:**
- IF blocks are C-shaped (open to accept blocks)
- THEN = if condition is true
- ELSE = if condition is false
- Player makes decisions based on conditions

**Try This:**
- What happens if defender is near?
- What happens if defender is not near?
- Can you add another IF block?

### Book 3: Loops (6 minutes)

**Your Task:** Create a loop for in & out dribble

**Steps - Manual First:**
1. Drag `START` block
2. Drag `FAKE LEFT` below START
3. Drag `GO RIGHT` below FAKE LEFT
4. Repeat 2 more times (6 blocks total)
5. Click RUN - see it work
6. Notice: Lots of blocks!

**Steps - Then Loop:**
1. Delete those blocks
2. Drag `REPEAT 3 TIMES` block
3. Drag `FAKE LEFT` inside the loop
4. Drag `GO RIGHT` inside the loop
5. Click RUN - see it work
6. Notice: Only 1 block!

**What You Learned:**
- Manual repetition = many blocks
- Loop = one block, repeats inside
- Loops are more efficient
- Same result, less code

**Try This:**
- Change the number (REPEAT 5 TIMES)
- What happens?
- Which is better - manual or loop?

### Anticipate Questions (3 minutes)

**Common Student Questions:**

**Q: "What if I get it wrong?"**
A: "That's okay! Debugging is part of coding. Try again, see what happens."

**Q: "Why does order matter?"**
A: "Just like giving directions - you can't turn left before you go straight. Code works the same way."

**Q: "Do I need to know Python?"**
A: "Not yet! We're using blocks first. You'll see Python later, but blocks are enough for now."

**Q: "What if I finish early?"**
A: "Great! Try creating your own sequence. Or help a classmate. Or try the Python code."

**Q: "What if I'm stuck?"**
A: "Ask a classmate. Or ask me. Or use the hints in the game. We're all learning together."

## ğŸ†˜ TROUBLESHOOTING GUIDE

### Technical Issues

**Game Won't Load:**
- Check internet connection
- Try different browser
- Clear cache
- Contact support

**Blocks Won't Snap:**
- Make sure you're dragging to code area
- Check if block fits (some blocks only fit in certain places)
- Try refreshing page

**Code Won't Run:**
- Check if START block is at top
- Make sure blocks are connected
- Look for error messages

### Student Issues

**Student Struggling:**
- Start simpler (fewer blocks)
- Pair with stronger student
- Provide hints, not answers
- Encourage persistence

**Student Finished Early:**
- Create own sequence
- Try Python code
- Help classmates
- Try different challenge

**Student Not Engaged:**
- Connect to their interests
- Let them create own play
- Celebrate small successes
- Make it a game
